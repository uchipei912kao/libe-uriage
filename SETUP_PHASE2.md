# 📲 フェーズ2 セットアップ手順（帰宅後 約10分）

外出先でもアプリを使えるように、**クラウドDB（Supabase）** に繋いで、**ネット上に公開**します。
コードは全部こちらで準備済みなので、下の手順を順番にやるだけでOKです。

---

## 全体の流れ
1. Supabaseの無料アカウントを作る（3分）
2. データの保存場所（テーブル）を作る（2分・コピペ1回）
3. カギ（URLとキー）をアプリに貼る（2分）
4. アプリをネットに公開する（3分）
5. iPhoneのホーム画面に追加（1分）

> 💡 何も設定しなくても、今のままアプリは動きます（スマホ1台の中だけで保存）。
> クラウド化すると「どの端末からでも同じデータが見られる」ようになります。

---

## ① Supabaseアカウントを作る

1. https://supabase.com を開く
2. 右上 **「Start your project」** → GitHubアカウントでログイン（既に持ってるGitHubでOK）
3. **「New project」** をクリック
   - Name（名前）：`libe-uriage` など何でもOK
   - Database Password：適当に決めて**メモしておく**（後で使う可能性あり）
   - Region：**Northeast Asia (Tokyo)** を選ぶ（日本が速い）
4. **「Create new project」** → 1〜2分待つ（DB準備中）

---

## ② テーブルを作る（コピペ1回）

1. 左メニューの **「SQL Editor」**（紙とペンのアイコン）を開く
2. **「New query」** をクリック
3. このフォルダの **`supabase_schema.sql`** の中身を**全部コピー**して貼り付け
4. 右下 **「Run」**（または ⌘+Enter）を押す
5. 「Success. No rows returned」と出ればOK ✅

これで `stock_items`（在庫）/ `sold_items`（売れたもの）/ `monthly_manual`（月別経費）の3つの保存場所ができました。

---

## ③ カギをアプリに貼る

1. Supabaseの左メニュー下の **「Project Settings」**（歯車）→ **「API」** を開く
2. 次の2つをコピーする：
   - **Project URL**（`https://xxxxx.supabase.co`）
   - **anon public** のキー（`eyJhbGci...` という長い文字列）
3. このフォルダの **`config.js`** を開いて、次のように貼る：

```js
const SUPABASE_URL = "https://xxxxx.supabase.co";   // ← Project URL を貼る
const SUPABASE_ANON_KEY = "eyJhbGci...";            // ← anon public を貼る
```

4. 保存する。これだけで、アプリが自動でクラウドモードに切り替わります。

> 💬 帰宅したらこの③だけ私（Claude）に「貼って」と言ってくれれば、こちらで貼り込みます。

---

## ④ ネットに公開する（Netlify Drop が一番ラク）

1. https://app.netlify.com/drop を開く
2. **このフォルダ（`リベ市場売上管理アプリ`）ごと** 画面にドラッグ＆ドロップ
3. 数十秒で `https://〇〇〇.netlify.app` というURLが発行される 🎉
4. そのURLをスマホで開けば、どこからでもアプリが使えます

> 別案：GitHub Pages でもOK（GitHubアカウントがあるので）。希望あれば切り替えます。

---

## ⑤ iPhoneのホーム画面に追加

1. iPhoneのSafariで ④ のURLを開く
2. 下の **共有ボタン**（□に↑）→ **「ホーム画面に追加」**
3. アイコン（青いグラフ）がホームに出る → アプリみたいに全画面で使える ✅

---

## ⚠️ セキュリティのメモ
- 今は個人利用前提で「anonキーで誰でも読み書き可」の設定です（シンプル運用）。
- 公開URLにキーが含まれるので、URLを他人に教えなければ実用上は問題ありません。
- もし「ログインしないと見られない」ようにしたくなったら、後から認証を足せます（言ってください）。
